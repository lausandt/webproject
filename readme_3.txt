Ik vind dat we moeten nadenken over wat we willen. Wat mij betreft zijn er 3 keuzes:

1 - We gaan door op deze hybride wijze van programmeren, dat betekent wat mij betreft we gaan verder met jouw code en ik kat de door mij gescgreven functionaliteit om. We implementeren MVC

2 - We gaan functioneel door. We slopen alle methoden eruit, we gebruik alleen immutables en geen objecten wel object-literals. We implementeren SAM

3 - We schrijven de code in reactieve stijl (zie index.js) en gebruiken REACT 

We kunnen dat laatste sowieso doen. ISM de haskell code zouden we dan 2 spelletjes hebben.
